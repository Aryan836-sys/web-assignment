Site Report – CSY1063 Web Development Assignment

1. Introduction

This website was developed as part of the CSY1063 Web Development module. With an emphasis on semantic structure, layout design, and responsive behaviour, the project aims to show my grasp on fundamental web technologies, particularly HTML and CSS. Throughout the module, the website serves as my personal portfolio, introducing me, showing my work, and my learning process.The development process was spread across several days, with the requirement to use only HTML and CSS, small improvements were made to the structure, layout, styling, and responsiveness .

2. Planning and Initial Setup

I used the assignment brief to plan the website's structure before I wrote any code. There are five primary pages on the website:

  • Home (index.html)
  • Projects (project.html)
  • Contact (contact.html)
  • Video Demonstration (videoDemo.html)
  • Site Report (sitereport.html)

All pages were designed with a consistent layout to guarantee professionalism and usability. This included same header, navigation bar, main content area, and footer across all of my pages.o keep files organised, a clear folder structure was first established.

The project uses the following structure:

  • HTML files stored in the root directory
  • A single external CSS file for styling
  • An image folder for images

The site is easier to maintain thanks to this structure.

3. HTML Structure and Semantics

Throughout the project, HTML is used to increase clarity and accessibility. Meaningful tags like <header>, <nav>, <main>, <section>, <article>, and <footer> were used in place of generic div elements.

  • The header contains the site logo and navigation menu
  • The navigation bar links all five pages
  • The main section holds page-specific content
  • The footer contains copyright information

Semantic elements make the code easier to read and guarantee that it is compatible with search engines and screen readers.

4. CSS Design and Layout

4.1 CSS Grid Layout

The primary layout method was CSS Grid for this project . To show adaptability and comprehension of grid-based design, various grid configurations were used on various pages.

  • Two-column grid layout on the Home page for the About section
  • Three-column grid layout for project and information cards
  • Single-column layouts on smaller screens

4.2 Spacing and Alignment

To ensure that all interactive elements are evenly spaced, particular attention was paid to the distance between buttons and navigation links. This enhances usability and prevents clutter. To make each section of the page feel purposeful, blank or unnecessary spaces were removed.

4.3 Typography and Colours

I chose simple fonts and colour schemes that don't stand out too much so that the design is clean and easy to read. To make it easier to read, there was a lot of contrast between the text and the background.

5. Home Page Design

  • A hero section with a welcome message and tagline
  • An About Me section describing my background and my interests
  • A section explaining what the website contains

The user can scroll through the page naturally because the content is organised into sections that are stacked vertically. This design is unique and suitable for an academic submission, but it draws inspiration from contemporary portfolio websites.

To improve user interaction, CSS was used to add hover effects and transitions.

Home Page initial design

6. Projects Page

To show versatility, the Projects page uses a different CSS Grid layout. Project cards with a title and description were added as placeholders. When the user interacts with specific projects, hover effects are used to visually highlight them.

Using media queries, the layout adjusts to various screen sizes, guaranteeing mobile device usability.

Projects Page initial design

7. Contact Page

  • Contact details
  • A functional email form using the mailto: method

Accessibility was considered by:

  • Using labels for all form inputs
  • Providing placeholders to guide the user
  • Ensuring proper spacing and readable form layout

Although no server-side processing is used, the form shows understanding of form structure and user interaction using HTML and CSS .

8. Responsive Design

  • Navigation links stacking vertically on mobile devices
  • Grid layouts switching from multiple columns to a single column
  • Font sizes and spacing adjusted for readability

As per the module requirement. All responsive behaviour was achieved by using CSS .

9. Problems Faced and Solutions

9.1 File Path Issues

Incorrect file paths were a major problem, particularly when linking images and CSS files. By realising that CSS paths are relative to the location of the CSS file and modifying paths appropriately, this issue was fixed.

9.2 Spacing and Alignment Problems

At first , some of the pages had uneven spacing and unwanted blank areas. This was fixed by refining grid layouts, margins, and padding values.

Use of AI Tools

During the development of this website,I used AI tools as a learning aid to support me in learning responsive design concepts and mobile usability. The AI assisted me on how CSS Grid, media queries, and layout adjustments work on smaller screen sizes.

AI was not used to generate the full website or copy complete solutions. Instead, it was used to clarify concepts, suggest improvements, and help troubleshoot layout and validation issues while I implemented and tested the final code myself.

In particular, AI guidance helped improve the mobile layout by:

  • Explaining how media queries adapt layouts for small screens
  • Improving spacing and alignment for mobile navigation
  • Identifying accessibility issues during validation

This made sure that the website remained within with the module requirement to use HTML and CSS while improving responsiveness and usability of the website.

10. Reflection and Learning Outcomes

  • Semantic HTML structure
  • CSS Grid and responsive layouts
  • Accessibility considerations

The project helped me in making me more comfortable with best practices in front-end development and responsive design with HTML and CSS.

11. Conclusion

This website successfully meets the requirements of the CSY1063 assignment. It shows a consistent design, clean structure, responsive behaviour, and thoughtful problem-solving. The development process was structured and repeated , allowing for continuous improvement while following the module constraints strictly.

Overall, this project shows my learning progress and understanding of web development principles.
